# Gemini R1｜前端 Shell 与视觉系统迭代要求

## 目标

按最新文档实现前端信息架构、Dashboard Shell、Editor Shell 和 Quiet App Builder 视觉系统。当前 Codex 负责后端与 Agent，Gemini 只处理前端 UI/UX。

## 必读资料

- `docs/DESIGN.md`
- `docs/02_PAGE_SPEC_页面说明文档.md`
- `docs/ui-atoms-cp/首页.png`
- `docs/ui-atoms-cp/编辑器-1.png`
- `docs/ui-atoms-cp/编辑器-2.png`
- `docs/ui-atoms-cp/编辑器-3.png`

## 实现范围

- 重建 `apps/web/src/App.tsx`、`styles.css`、`App.test.tsx`。
- 建立以下路由：
  - `/login`
  - `/dashboard`
  - `/projects`
  - `/templates`
  - `/resources`
  - `/app/:projectId`
  - `/app/:projectId/generating`
  - `/app/:projectId/inspect`
  - `/app/:projectId/versions`
  - `/app/:projectId/publish`
  - `/admin`
- Dashboard Shell 可使用安静左侧导航或顶部工作区导航，视觉参考 `首页.png` 和 `编辑器-2.png`。
- Editor Shell 禁止持久左侧导航，必须使用顶部栏：返回仪表盘、项目菜单、工具图标组、分享、更多、发布。
- 普通用户 UI 禁止出现 Codex、Docker、Vite、pnpm、dist、HMR、WebContainer、terminal、stdout、stderr、workspace 等技术词。
- 页面文案使用“应用查看器”“构建记录”“稳定版本”“预览快照”“继续修改”“一键发布”。

## 页面要求

- `/dashboard`：大 Prompt Composer、平台/风格/参考资料 chips、开始生成按钮、4 个示例卡片。
- `/projects`：项目卡片网格、搜索、状态筛选、排序、视图切换、新建应用按钮。
- `/templates`：精选模板 Hero、分类 tabs、模板卡片。
- `/resources`：已连接资源、推荐连接、右侧说明卡。
- `/app/:projectId`：左侧构建记录，右侧应用查看器，稳定快照预览框。
- `/app/:projectId/generating`：旧版本仍显示，新版本生成时间线。
- `/app/:projectId/inspect`：选择器微调面板、应用查看器、底部选中元素编辑面板。
- `/app/:projectId/versions`：版本时间线、版本详情、回退需二次确认 UI。
- `/app/:projectId/publish`：版本快照、发布设置、一键发布确认卡。
- `/admin`：可以显示技术状态，但保持简洁。

## 视觉要求

- 使用 `docs/DESIGN.md` token：背景 `#F8F8F6`、品牌蓝 `#315CF6`、弱边框 `#E7E8EC`。
- 卡片轻边框、轻阴影、16-20px 圆角。
- 使用 `lucide-react` 图标。
- 不使用大面积高饱和渐变、黑科技背景、终端风、IDE 风。
- 桌面优先，移动端至少无横向溢出。

## 接口约束

- 本轮可以使用前端静态 fixture/mock 数据呈现 UI。
- 不改后端 API。
- 不新增前端依赖，除非必须。
- 保留 `apps/web/src/api.ts`，不强制接入真实接口。

## 验收

- `pnpm --filter @atoms-cp/web typecheck`
- `pnpm --filter @atoms-cp/web test`
- `pnpm --filter @atoms-cp/web build`
- 测试至少覆盖：关键路由可渲染、Editor Shell 无持久侧边栏、普通用户页不出现禁用技术词。
