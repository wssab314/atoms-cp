# Gemini R4.1｜Inspector 微调前端闭环迭代要求

## 背景
Codex R4 已完成 Inspector/Selector 后端闭环：

- `GET /api/projects/:projectId/manifest`
- `GET /api/projects/:projectId/versions`
- `POST /api/projects/:projectId/selector/text-patch`

用户打开已生成项目后，可以在应用查看器中点选元素，看到元素信息，直接修改文案或输入 AI 修改要求，提交后进入构建/快照等待状态，并最终看到新的预览快照。

## 新后端接口契约

### Manifest
`GET /api/projects/:projectId/manifest`

返回：

```ts
{
  projectId: string;
  projectVersionId?: string;
  manifest: {
    entries: Record<string, {
      aiId: string;
      file: string;
      component: string;
      elementType: string;
      editable: Array<'text' | 'className' | 'styleTokens' | 'props'>;
      requirementId?: string;
    }>
  };
  entries: ManifestEntry[];
}
```

注意：普通用户页面不能显示 `file`、`component` 等技术字段原文。可以将其转译为“标题”“按钮”“段落”“区域”等产品化描述。

### Versions
`GET /api/projects/:projectId/versions`

返回 `ProjectVersionRecord[]`，按版本倒序。后端已经隐藏 `workspacePath`，前端也不能自行展示任何路径字段。

### Text Patch
`POST /api/projects/:projectId/selector/text-patch`

请求：

```ts
{
  aiId: string;
  text: string;
}
```

返回：

```ts
{
  manifestEntry: ManifestEntry;
  projectVersion: ProjectVersionRecord;
  files: ProjectFileRecord[];
  buildJob: BuildJobRecord;
  traceEvent?: TraceEventRecord;
}
```

### AI Patch
`POST /api/projects/:projectId/selector/ai-patch`

请求：

```ts
{
  aiId: string;
  instruction: string;
  selectedText?: string;
}
```

返回包含 `manifestEntry`、`projectVersion`、`buildJob`，并可包含 `agentRun`、`modelInvocation`。普通用户页面不能显示模型、provider、token、技术状态原文。

## Inspector iframe 消息协议

预览 iframe 需要进入 inspector 模式：

- 优先在 preview URL 上追加 `inspector=1`
- 或向 iframe `contentWindow` 发送 `{ type: 'INSPECTOR_ENABLE' }`

可发送命令：

```ts
{ type: 'INSPECTOR_ENABLE' }
{ type: 'INSPECTOR_DISABLE' }
{ type: 'INSPECTOR_HIGHLIGHT', aiId: string }
```

iframe 会向父窗口发送：

```ts
{
  type: 'atoms-cp:preview-element-selected' | 'atoms-cp:preview-element-hovered';
  event: 'INSPECTOR_SELECT' | 'INSPECTOR_HOVER';
  aiId: string;
  text?: string;
  className?: string;
  tagName: string;
}
```

前端只消费 `aiId`、`event`、`text`、`tagName`。不要展示 `className` 给普通用户。

## 实现范围

### 1. 扩展 `apps/web/src/api.ts`
补齐以下 API client：

- `fetchProjectManifest(projectId)`
- `fetchProjectVersions(projectId)`
- `patchSelectorText(projectId, input)`
- `patchSelectorAI(projectId, input)`

保留现有 fallback/mock 机制，但真实接口成功时必须使用真实数据。

### 2. 重建 `/app/:projectId/inspect` 数据流
页面进入时并发拉取：

- 项目详情
- manifest
- preview snapshots
- versions
- trace events

右侧应用查看器只加载 `active === true && status === 'ready'` 的快照。无 ready 快照时显示产品化骨架状态：“预览快照正在准备中”。

### 3. Inspector 交互
应用查看器 iframe 加载后启用 inspector。

收到 `INSPECTOR_HOVER`：

- 左侧/底部列表中轻量高亮对应元素
- 不弹出阻塞 UI

收到 `INSPECTOR_SELECT`：

- 设置当前选中元素
- 展示元素类型、当前文案、可编辑能力
- 如果 manifest 中不存在该 `aiId`，展示“该元素暂不支持修改”

从 manifest entry 推导普通用户文案：

- `heading`/`h1`/`h2` -> 标题
- `button` -> 按钮
- `paragraph`/`p` -> 段落
- `section` -> 区域
- 其他 -> 元素

### 4. 微调面板
选中元素后展示两个操作：

- “直接改文案”：输入框 + “应用修改”
- “让 AI 修改”：指令输入框 + “生成修改”

约束：

- 只有 `editable` 包含 `text` 才允许提交
- 空输入禁用提交
- 提交中显示“正在应用修改”
- 成功后显示“已开始生成新的预览快照”
- 失败时显示友好错误，不出现接口内部错误栈

### 5. Patch 后闭环
patch 成功后：

- 记录返回的 `projectVersion.id` 和 `buildJob.id`
- 跳转到 `/app/:projectId/generating`，或在 inspect 页内显示等待状态并轮询
- 轮询 snapshots，直到出现与新版本相关或最新 ready active snapshot
- ready 后刷新 iframe

优先沿用 R3.1 已实现的 generating 双条件逻辑，不重复造第二套复杂状态机。

### 6. Versions 页面接真实版本列表
`/app/:projectId/versions` 使用 `GET /versions` 的真实版本列表作为主数据源。

展示字段必须产品化：

- `initial_generate` -> 初始生成
- `selector_edit` -> 手动微调
- `agent_patch` -> AI 微调
- `code_edit` -> 代码修改
- `rollback` -> 版本回退

不要展示 `workspacePath`、文件路径、内部 source 原文。

## 视觉与体验要求

- 继续保持 Quiet App Builder 风格：浅背景、轻边框、克制阴影、品牌蓝点缀。
- Inspector 页仍然禁止持久左侧全局导航。
- iframe 区域必须占据主要视觉面积，微调面板不能喧宾夺主。
- 390px 移动视口不能横向溢出。
- 2 核 2G 小服务器约束下，前端轮询要克制：
  - 默认 3 秒一次
  - 页面不可见时暂停或降频
  - 失败后不要高频重试
- 不新增大型前端依赖。

## 禁止普通用户页面出现的词

普通用户路由禁止出现：

- Codex
- Docker
- Vite
- pnpm
- dist
- HMR
- WebContainer
- terminal
- stdout
- stderr
- workspace
- build job
- project version
- trace event
- manifest

可替代表达：

- project version -> 版本
- build job -> 构建任务 / 正在准备预览
- preview snapshot -> 预览快照
- manifest entry -> 可修改元素

Admin 页面可以显示技术状态，但仍需脱敏。
