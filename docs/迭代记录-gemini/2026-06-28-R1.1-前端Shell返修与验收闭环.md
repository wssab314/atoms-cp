- `/dashboard`：`scrollWidth=491`

- 390px 宽度下 `/dashboard` 不得横向溢出。

### 未通过项

#### P1｜390px 移动端 Editor 顶栏仍有可见越界

复现页面：

- `http://localhost:5178/app/demo`
- 视口：`390px × 844px`

Browser QA 观测结果：

- `.editor-header-right` 的可见区域从 `left=299` 延伸到 `right=471`，超出 390px 视口。
- `.btn.btn-primary` 发布按钮从 `left=391` 延伸到 `right=471`，基本被推出屏幕。
- 项目名 `销售数据看板` 被挤压成多行竖排显示。
- 页面整体 `scrollWidth` 仍等于 `clientWidth`，但这是因为 `html, body { overflow-x: hidden; }` 掩盖了横向滚动条；实际可交互元素仍发生了视口外裁切。

截图证据：

- `/tmp/atoms-cp-gemini-app-mobile.png`

### 问题原因

当前 Editor Shell 顶栏仍按桌面三段式结构布局：

- 左侧：返回仪表盘 + 项目选择器。
- 中间：工具图标组。
- 右侧：分享按钮 + 发布按钮。

在 390px 宽度下，这三组内容的最小宽度之和超过视口宽度。当前 CSS 仅对 Dashboard Shell 做了移动端响应式处理，但 Editor Shell 缺少同等强度的窄屏策略，因此出现：

- `.editor-header` 内部元素被强行压缩；
- `.project-selector-trigger` 文字缺少有效截断或折叠策略；
- `.editor-header-right` 保持完整文本按钮，无法在窄屏内收纳；
- `overflow-x: hidden` 只能隐藏横向滚动，不能解决按钮被裁切的问题。

这也是为什么现有自动化测试能通过：JSDOM 单元测试无法真实计算元素布局尺寸，且当前测试主要验证路由、文案和 DOM 约束，没有覆盖真实浏览器中的 `getBoundingClientRect()` 越界断言。

### 修复建议

建议 Gemini 做一次小范围 R1.2 返修，只处理 Editor Shell 移动端顶栏，不扩大重做页面。

推荐方案：

1. 在 `@media (max-width: 640px)` 下重排 `.editor-header`：
   - 允许顶栏变成两行布局；
   - 第一行保留返回入口、项目名、更多入口；
   - 第二行放置工具图标组；
   - 分享与发布收进更多菜单，或改成仅图标按钮。

2. 为项目选择器增加窄屏约束：
   - `.project-selector` 设置 `min-width: 0`；
   - `.project-selector-trigger span` 使用 `white-space: nowrap; overflow: hidden; text-overflow: ellipsis;`；
   - 在 390px 下不要让项目名竖排换行。

3. 为右侧操作区增加移动端折叠策略：
   - `.editor-header-right` 在窄屏下不要保留两个完整文字按钮；
   - 可改为 `display: none` 并由更多菜单承载；
   - 或保留发布图标按钮，隐藏 `分享`，按钮宽度控制在 40-44px。

4. 给真实浏览器 QA 增加布局断言：
   - 在 390px 宽度下检查 `.editor-header *` 中不存在 `rect.right > window.innerWidth + 1`；
   - 同时检查 `.editor-header *` 中不存在 `rect.left < -1`；
   - 单元测试仍可保留文案/路由/禁用词检测，但不要把它当成移动端布局验收的唯一依据。
