# 02｜页面说明文档｜极简工作台版

> 文档用途：定义 Result-first AI App Builder 的 P0/P1 页面结构、布局、状态、交互与验收标准。
> 当前设计方向：**极简工作台风格**，借鉴 Atoms 的安静工作台气质与顶部项目菜单逻辑，但不复刻其品牌、信息密度或开发者工具表达。
> 技术基线：P0 采用 **宿主机持久化 workspace + Docker Codex Worker + build dist + 自有 Preview Service 静态快照预览**。P0 不使用 WebContainer、不使用 HMR、不为每个用户启动 Vite dev server、不展示代码编辑器或 terminal。

---

## 1. 产品页面原则

### 1.1 三个差异化原则

| 原则 | 页面表达 |
|---|---|
| 少开发者工具感 | 默认不展示代码、终端、WebContainer、HMR、npm、构建日志原文；只在“构建详情”中折叠展示必要技术信息。 |
| 强调稳定版本快照 | 所有生成结果都以稳定版本保存。构建新版本时继续展示旧版本预览，新版本通过校验后再切换。 |
| 面向非技术用户 | 使用“应用查看器”“构建记录”“继续修改”“版本历史”“一键发布”等业务语言，避免 IDE 心智。 |

### 1.2 用户默认心智

用户进入产品后应理解：

> 我描述一个应用目标，AI 会生成一个可预览的稳定应用版本；我可以继续微调、回退版本或发布上线。

用户不应被迫理解：

- React / Vite / pnpm / build / dist；
- WebContainer / HMR / terminal；
- 文件树、源码结构、运行时细节。

## 2. 页面信息架构

### 2.1 P0 路由建议

| 路由 | 页面 | P0/P1 | 说明 |
|---|---|---:|---|
| `/login` | 登录页 | P0 | 邮箱验证码 / OAuth 登录。 |
| `/dashboard` | 仪表盘 / 创建入口 | P0 | 默认入口，聚焦大输入框与最近项目。 |
| `/projects` | 我的项目 | P0 | 项目列表、状态、继续修改、预览、发布。 |
| `/templates` | 模板中心 | P0 | 官方模板和示例应用。 |
| `/resources` | 资源中心 | P0 | 连接 GitHub / Vercel / Supabase / 文件 / 链接等资源。 |
| `/app/:projectId` | 项目工作台 | P0 | 构建记录 + 应用查看器。 |
| `/app/:projectId/generating` | 生成新版本状态 | P0 | 快照式构建过程，保留旧版本预览。 |
| `/app/:projectId/inspect` | 选择器微调 | P0 | 点击页面元素后提出局部修改。 |
| `/app/:projectId/versions` | 版本历史 | P0 | 稳定版本快照、预览、回退、对比。 |
| `/app/:projectId/publish` | 版本与发布 | P0 | 当前版本、发布设置、一键发布。 |
| `/admin` | 管理后台 | P1 | Worker、Job、用户、构建失败、系统配置。 |

### 2.2 页面 Shell 分类

| Shell | 使用页面 | 布局规则 |
|---|---|---|
| Dashboard Shell | `/dashboard`、`/projects`、`/templates`、`/resources` | 顶部品牌栏 + 工作区菜单 + 主内容区。可以有轻量导航，但不强依赖固定侧边栏。 |
| Editor Shell | `/app/:projectId/**` | **无持久侧边栏**。顶部提供“返回仪表盘”按钮、项目菜单、工具按钮、分享、发布。 |
| Admin Shell | `/admin` | 可使用传统侧边栏，因为用户是运营/管理员。 |

## 3. Editor Shell：项目编辑器全局布局

### 3.1 顶部栏

项目编辑器中不再展示左侧持久导航。顶部栏承担导航和项目管理入口。

```text
┌─────────────────────────────────────────────────────────────┐
│ ← 返回仪表盘  [项目名称 ▾]        工具图标组        分享  ⋯  发布 │
└─────────────────────────────────────────────────────────────┘
```

### 3.2 顶部元素说明

| 区域 | 内容 | 说明 |
|---|---|---|
| 返回按钮 | `返回仪表盘` | 回到 `/dashboard` 或 `/projects`。 |
| 项目菜单 | `销售数据看板 ▾` | 展开项目菜单。 |
| 中央工具组 | 移动端/桌面端、网格、历史、反馈、信息等图标 | 图标必须少，P0 不超过 6 个。 |
| 右侧操作 | 分享、更多、发布 | 发布按钮为主动作。 |

### 3.3 项目菜单内容

项目名下拉菜单用于替代编辑器侧边栏。

| 菜单项 | P0/P1 | 说明 |
|---|---:|---|
| 项目信息与设置 | P0 | 项目名称、描述、目标平台。 |
| 连接器管理 | P0 | 进入资源/连接器设置。 |
| 克隆项目 | P1 | 复制项目为新项目。 |
| 收藏项目 | P1 | 加入常用项目。 |
| 外观设置 | P0 | 切换主题、密度、字体风格。 |
| 帮助中心 | P0 | 打开帮助文档。 |

### 3.4 Editor 主体通用布局

```text
┌─────────────────────┬───────────────────────────────────────┐
│ 构建记录 / 微调记录  │ 应用查看器 / 版本快照 / 发布设置        │
│ AI 摘要 / 建议       │ iframe Snapshot Preview / 设置卡片     │
└─────────────────────┴───────────────────────────────────────┘
```

| 区域 | 建议宽度 | 职责 |
|---|---:|---|
| 左侧任务面板 | 320-420px | 解释 AI 做了什么、展示构建步骤、给出下一步建议。 |
| 右侧主区域 | 自适应 | 展示应用预览、版本、发布、资源设置等结果。 |

## 4. 页面 1：登录页

### 4.1 目标

完成用户认证，并让用户进入自己的工作区。

### 4.2 结构

```text
居中登录卡片
  Logo / 产品名
  标题：登录灵感工坊
  邮箱输入
  验证码输入 / OAuth 登录
  协议说明
```

### 4.3 状态

| 状态 | 页面表现 |
|---|---|
| 默认 | 邮箱输入、获取验证码按钮。 |
| 验证码已发送 | 验证码输入框、倒计时、重新发送。 |
| 登录失败 | 卡片内轻量错误提示。 |
| 登录成功 | 跳转 `/dashboard`。 |

### 4.4 验收

- 登录卡片居中，页面无多余营销信息。
- 错误提示明确，不出现技术错误栈。

## 5. 页面 2：仪表盘 / 创建入口

### 5.1 目标

让用户通过自然语言开始创建应用。

### 5.2 结构

```text
顶部品牌栏
  Logo / 工作区菜单 / 通知 / 用户菜单

主区域
  轻量流程提示：描述目标 → 生成应用 → 预览发布
  主标题：你想创建什么应用？
  副标题：用自然语言描述你的想法，AI 将为你生成可用的应用结果
  Prompt Composer
  示例模板卡片
  最近项目入口
```

### 5.3 Prompt Composer

| 元素 | 说明 |
|---|---|
| 多行输入框 | placeholder：`描述你的应用想法，例如：一个帮助小型团队管理项目进度的看板应用...` |
| 平台选择 | P0 默认 `Web App / H5`。 |
| 风格选择 | 默认 `极简工作台`。 |
| 参考资料 | 上传文档、图片、链接。 |
| 开始生成 | 主按钮，触发创建项目与生成任务。 |

### 5.4 示例卡片

建议展示 4 个：

- 销售数据看板；
- 客户反馈收集；
- 活动报名系统；
- 知识库中心。

### 5.5 交互

| 操作 | 结果 |
|---|---|
| 点击“开始生成” | 创建 project/version/job，进入生成状态页。 |
| 点击示例卡片 | 自动填充 prompt 或从模板创建。 |
| 上传参考资料 | 进入资源解析状态，成功后绑定到新项目。 |

### 5.6 空 / 错误状态

| 状态 | 文案 |
|---|---|
| prompt 为空 | `请先描述你想创建的应用。` |
| prompt 过短 | `请补充应用目标、核心功能或目标用户。` |
| 额度不足 | `当前额度不足，无法开始生成。` |

### 5.7 验收

- 页面第一视觉中心是输入框。
- 不出现代码、终端、WebContainer、技术标签。
- 用户 5 秒内知道如何开始。

## 6. 页面 3：我的项目

### 6.1 目标

集中管理用户已创建的应用项目。

### 6.2 结构

```text
Top Bar
  返回仪表盘 / 工作区菜单 / 品牌 / 通知 / 用户

主内容
  标题：我的项目
  Tabs：全部项目 / 我创建的 / 最近打开
  Toolbar：搜索 / 状态筛选 / 更新时间排序 / 视图切换 / 新建应用
  Project Cards Grid
  Pagination
```

### 6.3 项目卡片字段

| 字段 | 示例 | 说明 |
|---|---|---|
| 图标 | 数据柱状图 | 依据应用类型生成。 |
| 名称 | 销售数据看板 | 项目名。 |
| 描述 | 销售核心指标实时监控与趋势分析看板 | 80 字以内。 |
| 状态 | 构建中 / 已完成 / 待发布 / 已发布 / 失败 | 使用用户可理解语言。 |
| 更新时间 | 今天 14:32 | 最近修改时间。 |
| 稳定版本数 | 1 / 3 / 4 | 对应 preview snapshots。 |
| 操作 | 继续修改 / 预览 / 发布 / 更多 | 主次分明。 |

### 6.4 状态标签

| 状态 | UI 表现 |
|---|---|
| 构建中 | 蓝色点 + `构建中`。 |
| 已完成 | 绿色点 + `已完成`。 |
| 待发布 | 橙色点 + `待发布`。 |
| 已发布 | 蓝色或绿色 badge + `已发布`。 |
| 失败 | 红色弱提示 + `需要处理`。 |

### 6.5 交互

| 操作 | 结果 |
|---|---|
| 继续修改 | 进入 `/app/:projectId`。 |
| 预览 | 打开当前稳定快照。 |
| 发布 | 进入 `/app/:projectId/publish`。 |
| 更多 | 重命名、复制、归档、删除。 |
| 新建应用 | 返回创建入口或打开新建弹窗。 |

### 6.6 验收

- 卡片数量较多时仍保持低噪音。
- 状态一眼可辨。
- “继续修改”永远是项目卡的主要操作。

```text

主区域
  已连接资源
  推荐连接

右侧说明卡
  资源如何提升应用生成效果
```

### 8.3 已连接资源

建议 P0 支持展示：

| 资源 | 说明 | 操作 |
|---|---|---|
| GitHub | 同步代码仓库、文档与 Issues | 管理 |
| Supabase | 连接数据库与身份认证 | 管理 |
| 飞书 | 获取群聊、文档与工作流数据 | 管理 |

| 资源 | 说明 |
|---|---|
| Vercel | 部署与托管应用。 |
| 上传文件 | 上传文档、数据或图片作为知识源。 |

文案方向：

### 8.6 交互

| 操作 | 结果 |
|---|---|
| 连接 | 打开 OAuth / 配置弹窗。 |
| 管理 | 查看权限、同步状态、断开连接。 |

### 8.7 验收

- 所有连接都以“对应用生成有什么帮助”解释，而不是纯技术连接器列表。

## 9. 页面 6：项目工作台 / 应用查看器

### 9.1 目标

展示某个项目的当前稳定版本，并提供继续修改、分享、发布入口。

### 9.2 结构

```text
Top Bar
  返回仪表盘 / 项目名称菜单 / 工具图标 / 分享 / 更多 / 发布

主区域
  左：构建记录
  右：应用查看器
```

### 9.3 构建记录面板

#### 9.3.1 内容结构

```text
构建记录
  AI 摘要卡
  已完成的步骤
  下一步你可以尝试
  本次构建信息
```

#### 9.3.2 用户可见步骤

| 步骤 | 文案 |
|---|---|
| 已理解目标 | 分析需求，明确核心功能与关键指标。 |
| 已生成页面结构 | 设计主要模块布局与页面流程。 |
| 已完成数据对接 | 接入所需数据或 mock 数据。 |
| 已完成图表与可视化 | 配置图表、列表、卡片等展示模块。 |
| 已生成预览快照 | 生成稳定应用预览，准备供你查看。 |

#### 9.3.3 下一步建议

建议以 chip 展示：

- 添加同比/环比对比；
- 添加筛选条件；
- 调整图表配色；
- 导出报表功能；
- 优化移动端布局；
- 更多建议。

### 9.4 应用查看器

| 元素 | 说明 |
|---|---|
| 标题 | `应用查看器`。 |
| 设备切换 | `移动端 / 桌面端`。 |
| 刷新 | 重新加载 Preview Service iframe。 |
| 当前版本 | `当前版本 v1.0`。 |
| iframe | 加载静态 snapshot preview URL。 |

### 9.5 预览行为

- 预览来自 `Preview Service`，加载稳定 `dist` 快照。
- 页面不展示开发中半成品。
- 如果用户继续修改，新版本未完成前继续展示旧版本。

### 9.6 验收

- 用户能明显看到“结果”，而不是“代码过程”。
- 所有技术细节默认折叠。
- `发布` 按钮始终在右上可见。

## 10. 页面 7：生成新版本状态

### 10.1 目标

在新版本生成过程中保持用户信任，并避免展示不稳定中间态。

### 10.2 核心规则

> 构建新版本时，应用查看器继续显示当前稳定版本；新版本通过校验并生成快照后再切换。

### 10.3 结构

```text
Top Bar

主标题：正在生成新版本...
副标题：我们采用快照发布机制，完成后将生成版本 2，新版本就绪后可随时预览

左：构建记录时间线
右：应用查看器（当前显示：版本 1）
右上：预计剩余时间 / 队列位置
```

### 10.4 时间线步骤

| 状态 | 文案 |
|---|---|
| completed | 正在理解修改需求 / 完成 / 时间。 |
| completed | 正在更新页面结构 / 完成 / 时间。 |
| active | 正在生成新版本 / AI 正在生成页面内容与样式。 |
| pending | 正在校验构建 / 完成质量校验与兼容性检查。 |
| pending | 正在准备预览快照 / 生成版本 2 的预览快照。 |

### 10.5 应用查看器提示

Banner 文案：

> 正在生成版本 2，当前显示版本 1 的稳定预览。新版本生成完成后，你将收到通知并可立即预览。

### 10.6 失败状态

| 失败类型 | 用户文案 | 操作 |
|---|---|---|
| 需求理解失败 | `没有理解清楚修改目标，请补充说明。` | 重新描述 |
| 构建失败 | `新版本没有通过检查，当前稳定版本不受影响。` | 自动重试 / 查看原因 |
| 超时 | `生成耗时较长，已保留当前稳定版本。` | 稍后查看 / 取消 |

### 10.7 验收

- 用户始终看到可用旧版本。
- 不出现空白 iframe 或半成品页面。
- 失败不影响当前版本。

## 11. 页面 8：选择器微调

### 11.1 目标

让用户通过点击预览中的页面元素，提出局部修改需求。

### 11.2 结构

```text
Top Bar

左：选择器微调面板
  微调记录
  AI 建议
  修改建议 chips
  自然语言输入

右：应用查看器
  稳定预览
  选中元素高亮
  底部选中元素编辑面板
```

### 11.3 选中元素表现

| 元素 | UI 表现 |
|---|---|
| 未选中 | 预览正常展示。 |
| hover | 弱蓝色描边。 |
| selected | 蓝色 2px 描边 + `已选择` badge。 |
| locked | 灰色虚线描边，不允许修改。 |

### 11.4 左侧微调记录

记录类型：

- 正在调整所选元素；
- AI 建议：卡片视觉层级；
- AI 建议：按钮样式优化；
- 已提交修改；
- 修改已生成新版本。

### 11.5 快速建议 chips

| 类型 | 示例 |
|---|---|
| 层级 | 按钮更突出、强调主标题。 |
| 空间 | 减小卡片间距、增加留白。 |
| 视觉 | 调整配色、统一圆角、增加阴影。 |
| 内容 | 文案更简洁、数字更醒目。 |

### 11.6 底部选中元素面板

字段：

| 字段 | 示例 |
|---|---|
| 元素名称 | 销售数据卡片。 |
| 元素类型 | 容器卡片 / 按钮 / 标题 / 图表。 |
| 位置 | 概览 / 核心指标 / 销售额卡片。 |
| 修改指令 | 自然语言输入框。 |
| 快速指令 | 数值更大更醒目、改为品牌蓝色、增加卡片阴影。 |

### 11.7 交互流程

```text
用户点击预览元素
  → 系统识别 selector path
  → 左侧展示当前选中元素和建议
  → 用户输入修改要求
  → 创建局部 change request
  → 进入生成新版本状态
  → 构建成功后切换新快照
```

### 11.8 验收

- 用户不需要理解 DOM、CSS selector。
- 局部修改不会破坏整个项目布局。
- 未完成构建前旧版本仍可查看。

## 14. 页面 11：构建失败 / 修复中状态

### 14.1 目标

构建失败时提供稳定反馈，不让用户接触技术噪音。

### 14.2 页面表现

```text
左：构建记录
  当前版本仍可用
  新版本未通过检查
  正在尝试自动修复 / 已停止

右：应用查看器
  继续显示上一稳定版本
```

### 14.3 文案

| 情况 | 文案 |
|---|---|
| 自动修复中 | `新版本没有通过检查，我们正在尝试自动修复。当前稳定版本不受影响。` |
| 自动修复失败 | `新版本暂时无法生成。你可以调整需求后重试，当前版本仍可继续使用。` |
| 用户可查看详情 | `查看构建详情`，默认折叠。 |

### 14.4 技术详情折叠项

仅在用户点击后展示：

- 错误摘要；
- 失败阶段；
- 重试次数；
- 支持复制错误报告。

不得默认展示完整 terminal output。

## 20. 总体验收标准

1. 编辑器内无持久侧边栏，使用顶部项目菜单和返回仪表盘按钮。
2. 主流程不出现代码编辑器、terminal、WebContainer、HMR。
3. 预览始终来自稳定 snapshot，不展示未完成中间态。
4. 新版本生成失败不影响当前稳定版本。
5. 页面文案面向非技术用户。
6. 用户能从任意项目页快速完成：继续修改、预览、查看版本、发布。
7. 所有生成/修改都可追溯到版本历史。
