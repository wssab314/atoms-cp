# DESIGN.md｜Quiet App Builder Design System

> 适用范围：Result-first AI App Builder 全部用户端页面。
> 设计方向：**Atoms-like Quiet Workspace，但做三处差异化**：更少开发者工具感、更强调稳定版本快照、更面向非技术用户。
> 视觉关键词：极简、安静、留白、纸面感、轻卡片、低技术感、结果优先、可发布感。

---

## 1. 设计原则

### 1.1 Result-first

界面始终围绕“可见结果”组织，而不是围绕代码组织。

- 默认展示应用结果、构建记录、版本快照。
- 不默认展示源码、terminal、构建日志原文。
- 技术细节只在“构建详情”或 Admin 中折叠展示。

### 1.2 Snapshot-first

P0 采用静态快照预览，不做 HMR 式实时开发体验。

- 每次生成/修改都形成一个稳定版本。
- 新版本构建期间保留旧版本预览。
- 新版本通过校验后才切换预览。
- 构建失败不影响当前稳定版本。

### 1.3 Non-technical first

页面语言面向非技术用户。

| 不推荐 | 推荐 |
|---|---|
| build dist | 准备预览快照 |
| Codex 修改代码 | 正在生成页面内容 |
| pnpm build failed | 新版本未通过检查 |
| dev server | 应用查看器 |
| deploy pipeline | 发布流程 |

### 1.4 Calm confidence

界面要给用户“稳定、可信、可控”的感受。

- 颜色低饱和；
- 留白充足；
- 不使用强烈霓虹、黑科技背景、大面积渐变；
- 不展示过度复杂的工程状态。

## 2. 信息架构与布局 Shell

### 2.1 Dashboard Shell

用于仪表盘、项目列表、模板中心、资源中心。

```text
Top Bar
  Logo / 工作区菜单 / 搜索或上下文操作 / 用户菜单

Main Content
  页面标题
  筛选或主操作
  卡片或列表内容
```

### 2.2 Editor Shell

用于项目工作台、生成状态、选择器微调、版本历史、版本与发布。

**编辑器内不展示持久侧边栏。**

```text
┌──────────────────────────────────────────────────────────────┐
│ ← 返回仪表盘  [项目名称 ▾]       工具图标组        分享  ⋯  发布 │
├──────────────────────────────────────────────────────────────┤
│ Main Workspace                                                │
└──────────────────────────────────────────────────────────────┘
```

### 2.3 Editor 主体布局

常规项目工作台：

```text
┌─────────────────────┬──────────────────────────────────────┐
│ 构建记录             │ 应用查看器                            │
│ AI 摘要 / 建议        │ 稳定 Snapshot Preview iframe          │
└─────────────────────┴──────────────────────────────────────┘
```

选择器微调：

```text
┌─────────────────────┬──────────────────────────────────────┐
│ 微调记录 / 建议       │ 应用查看器 + 选中元素 + 底部编辑面板   │
└─────────────────────┴──────────────────────────────────────┘
```

版本历史：

```text
┌────────────────────────────────┬─────────────────────────────┐
│ 版本时间线                      │ 当前选中版本详情              │
└────────────────────────────────┴─────────────────────────────┘
```

## 3. 视觉基调

### 3.1 风格名称

**Quiet App Builder / 安静应用工作台**

### 3.2 情绪目标

| 目标 | 表现 |
|---|---|
| 安静 | 大面积留白、弱边框、低阴影。 |
| 专业 | 规范网格、稳定控件、清晰状态。 |
| 可信 | 版本、保存、回退、发布状态明确。 |
| 不压迫 | 不展示复杂技术术语，不堆叠密集工具栏。 |

### 3.3 不要做成

- 在线 IDE；
- 开发者控制台；
- Terminal-first 产品；
- 黑底科技风；
- 强游戏化生成器；
- 复杂 SaaS 后台模板。

---

## 3.4 标准 DesignProfile 模板

生产环境固定提供 5 个标准设计模板。模板用于把 `AppSpec` 转成稳定的视觉方向，不使用临时 mock 或随机方案。

| 模板 ID | 名称 | 适用场景 | 视觉重点 |
|---|---|---|---|
| `quiet-personal` | Quiet Personal | 个人网站、作品集、简历、个人品牌页 | 身份识别、代表作品、经历与联系方式。 |
| `studio-minimal` | Studio Minimal | 服务介绍、专家主页、预约转化 | 信任感、服务说明、单一清晰行动入口。 |
| `product-launch` | Product Launch | 产品官网、SaaS、活动报名、Waitlist | 产品价值、功能证明、转化路径。 |
| `operator-dashboard` | Operator Dashboard | 数据看板、管理台、内部工具 | 扫描效率、筛选、表格和状态清晰度。 |
| `content-hub` | Content Hub | 博客、知识库、资源中心、学习内容 | 阅读层级、分类导航、资源组织。 |

默认排序由用户需求和 `AppSpec` 内容决定。比如“个人网站/作品集”优先 `quiet-personal`，“预约/咨询/课程”优先 `studio-minimal`，“管理/看板/数据”优先 `operator-dashboard`。

## 7. 组件规范

### 7.1 Top Bar

#### 用途

全局导航、项目上下文和主操作。

#### 结构

```text
Left: 返回仪表盘 / 项目菜单
Center: 工具图标组
Right: 分享 / 更多 / 发布
```

#### 规则

- 高度：64-72px。
- 背景：白色或半透明白。
- 底部边框：1px `--color-border`。
- 中央工具图标 P0 不超过 6 个。
- `发布` 始终是最高优先级按钮。

### 7.2 Project Menu

#### 内容

- 项目信息与设置；
- 连接器管理；
- 克隆项目；
- 收藏项目；
- 外观设置；
- 帮助中心。

#### 视觉

- 宽度 220-280px。
- 圆角 `--radius-lg`。
- 阴影 `--shadow-popover`。
- 每项高度 40-44px。
- 图标弱灰，文字深色。

### 7.3 Prompt Composer

#### 用途

首页创建应用的核心输入组件。

#### 结构

```text
Textarea
Option chips: 平台 / 风格 / 参考资料
Primary CTA: 开始生成
```

#### 规则

- 宽度 720-960px。
- 高度 180-260px。
- 圆角 `--radius-2xl`。
- CTA 固定在右下区域。
- 输入为空时按钮 disabled。

### 7.4 Card

#### 类型

| 类型 | 用途 |
|---|---|
| Project Card | 我的项目列表。 |
| Template Card | 模板中心。 |
| Resource Card | 资源中心。 |
| Version Card | 版本历史 / 发布。 |
| Status Card | 构建状态 / 发布状态。 |

#### 规则

- 背景白色。
- 边框 1px。
- 圆角 16-20px。
- hover 可提升边框或轻阴影，不做强动效。

### 7.5 Button

| 类型 | 视觉 | 用途 |
|---|---|---|
| Primary | 品牌蓝底白字 | 开始生成、一键发布、发布。 |
| Secondary | 白底灰边 | 预览、分享、继续修改。 |
| Ghost | 无背景 | 弱操作、工具栏图标。 |
| Danger | 红色弱背景 | 删除、断开连接。 |

按钮高度：

- 小：32px；
- 中：40px；
- 大：48px。

### 7.6 Badge / Status

| 状态 | 文案 | 颜色 |
|---|---|---|
| building | 构建中 | 蓝色 |
| done | 已完成 | 绿色 |
| pending_publish | 待发布 | 橙色 |
| published | 已发布 | 绿色或蓝色 |
| failed | 需要处理 | 红色 |
| current | 当前版本 | 蓝色 |
| history | 历史版本 | 灰蓝 |

### 7.7 Application Viewer

#### 用途

展示 Preview Service 的稳定快照。

#### 结构

```text
Header
  标题：应用查看器
  设备切换：移动端 / 桌面端
  刷新
  当前版本

Content
  iframe snapshot preview
```

#### 规则

- 不显示 dev server 信息。
- 不显示 HMR 状态。
- 构建中继续展示旧版本。
- iframe 外层使用白色大卡片。
- 预览 frame 可以有轻阴影和圆角。

### 7.8 Build Record

#### 用户可见步骤

- 已理解目标；
- 已生成页面结构；
- 已完成构建；
- 已生成预览快照。

#### 规则

- 默认显示自然语言摘要。
- 不显示 terminal output。
- 失败时显示“查看构建详情”折叠项。

### 7.9 Selector Tuning Panel

#### 状态

| 状态 | 表现 |
|---|---|
| hover | 弱描边。 |
| selected | 蓝色描边 + `已选择`。 |
| editing | 底部编辑面板展开。 |
| submitted | 进入生成新版本。 |

#### 底部面板内容

- 已选中元素；
- 元素类型；
- 元素路径；
- 修改指令输入框；
- 快速建议；
- 使用提示。

### 7.10 Version Timeline

#### 规则

- 当前版本卡使用品牌蓝边框。
- 历史版本低强调。
- 回退、设为当前属于敏感操作，需二次确认。
- 每个版本必须有摘要和快照缩略图。

## 8. 页面级视觉要求

### 8.1 仪表盘 / 创建入口

- 主视觉居中。
- 大标题可以使用更有编辑感的 serif-like 字体。
- Prompt Composer 是绝对视觉中心。
- 示例卡片不超过 4 个。

### 8.2 我的项目

- 卡片网格清晰。
- 最近打开项目可以弱蓝描边。
- 状态与操作区域要固定位置，避免卡片参差。

### 8.3 模板中心

- 顶部推荐模板可以使用大横幅。
- 模板缩略图必须是“成品应用感”，不是代码截图。
- 使用模板按钮为主操作。

### 8.4 资源中心

- 强调“资源能提升生成质量”。
- 连接状态要清楚。
- 资源卡片不要像开发者 API 控制台。

### 8.5 项目工作台

- 左侧构建记录不超过 420px。
- 右侧应用查看器占主要宽度。
- 顶部项目菜单替代侧边栏。

### 8.6 生成新版本

- 显示旧版本仍稳定可用。
- 时间线使用用户可理解阶段。
- 不展示半成品 preview。

### 8.7 选择器微调

- 选中元素描边清晰但不遮挡内容。
- 底部编辑面板不能过高，避免遮挡大部分预览。
- 快捷建议要短，最多两行。

### 8.8 版本历史

- 左侧时间线，右侧详情。
- 当前版本最高强调。
- 变更摘要要分组：新增、优化、修复。

### 8.9 版本与发布

- 用户必须明确“当前要发布的是哪个版本”。
- 发布设置以业务语言解释。
- 一键发布按钮只在必要连接完成后可用。

## 9. 交互规范

### 9.1 生成应用

```text
用户输入 prompt
  → 创建项目
  → 进入生成状态
  → 保持阶段进度
  → 构建成功
  → 生成稳定快照
  → 切换到项目工作台
```

### 9.2 继续修改

```text
用户输入修改需求
  → 创建新 version
  → 当前预览继续展示旧 version
  → 新 version 构建成功
  → 提示用户切换或自动切换
```

### 9.3 选择器微调

```text
点击元素
  → 高亮选中
  → 展示元素信息和建议
  → 输入修改指令
  → 创建局部修改任务
  → 生成新版本
```

### 9.4 发布

```text
选择当前版本
  → 检查发布配置
  → 一键发布
  → 发布校验
  → 返回线上 URL
```

### 9.5 回退

```text
选择历史版本
  → 点击回退
  → 二次确认
  → 创建 rollback 记录
  → 当前版本切换为历史 snapshot
```

## 10. 状态设计

### 10.1 Loading

| 场景 | 组件 |
|---|---|
| 首页生成 | 阶段进度 + 文案。 |
| 项目列表加载 | Skeleton Card。 |
| 应用预览加载 | Viewer Skeleton。 |
| 资源连接中 | 行内 spinner。 |

### 10.2 Empty

| 页面 | 文案 |
|---|---|
| 我的项目 | 还没有项目。描述一个业务想法，生成第一个可预览应用。 |
| 模板搜索无结果 | 没有找到合适模板。你可以提交需求，我们帮你生成。 |
| 资源为空 | 还没有连接资源。连接文档、数据或工具，可以提升生成质量。 |
| 版本历史为空 | 还没有稳定版本。完成第一次构建后会自动保存。 |

### 10.3 Error

错误必须遵守：

- 先说明影响范围；
- 再说明用户能做什么；
- 技术详情折叠。

示例：

> 新版本没有通过检查，当前稳定版本不受影响。你可以调整需求后重试，或展开查看原因。

### 10.4 Success

成功提示要短：

- 已生成预览快照；
- 构建成功；
- 发布成功；
- 已切换当前版本。

## 11. 文案规范

### 11.1 语气

- 平静、明确、可信；
- 不夸张、不营销化；
- 不把失败包装成成功；
- 不使用过多感叹号。

### 11.2 推荐文案

| 场景 | 文案 |
|---|---|
| 首页标题 | 你想创建什么应用？ |
| 首页副标题 | 用自然语言描述你的想法，AI 将为你生成可用的应用结果。 |
| 构建中 | 正在生成新版本... |
| 快照说明 | 当前显示上一稳定版本，新版本就绪后会自动更新。 |
| 失败说明 | 新版本未通过检查，当前稳定版本不受影响。 |
| 发布说明 | 发布后，用户即可访问你的应用。所有结果均已保存，可随时回滚。 |

### 11.3 禁用默认暴露词

普通用户端默认不出现：

- Codex；
- Docker；
- Vite；
- pnpm；
- dist；
- HMR；
- WebContainer；
- terminal；
- stdout；
- stderr；
- stack trace。

## 12. 可访问性与响应式

### 12.1 可访问性

- 所有按钮需要清晰 focus ring。
- 主按钮与背景对比度满足可读性。
- 状态不能只靠颜色表达，必须配文字。
- 菜单可键盘导航。
- iframe 加载失败需要有文本 fallback。

### 12.2 桌面断点

| 宽度 | 布局 |
|---|---|
| ≥1440px | 标准双栏工作台。 |
| 1200-1439px | 左侧面板收窄，右侧预览优先。 |
| 960-1199px | 左侧可折叠，预览优先。 |
| <960px | 改为单列，先预览后记录；P0 仅基础适配。 |

### 12.3 移动端

P0 移动端只保证：

- 查看项目列表；
- 查看预览；
- 查看版本；
- 简单发布状态。

不要求完整编辑体验。

## 13. 实现注意事项

### 13.1 前端实现

- 组件优先按 Card / Panel / TopBar / Viewer / Timeline / Badge 组织。
- 不要在多个页面重复手写状态样式。
- 应用查看器 iframe 外层必须独立组件化。
- 版本状态、构建状态、发布状态应统一映射。

### 13.2 Preview Service UI 约束

- iframe `src` 只能来自已授权 preview URL。
- 构建中不切换到未完成版本。
- 快照加载失败时展示错误卡片，不展示浏览器默认错误页。

### 13.3 给 Codex 的 UI 修改约束

当让 Codex 修改前端时，应明确：

- 不新增 terminal/code editor 入口；
- 不新增 WebContainer/HMR 文案；
- 不把构建日志直接放到普通用户界面；
- 不改变 Editor Shell 的“无侧边栏”结构；
- 不绕过版本快照流程展示中间态。

---

## 14. 禁止事项

1. 编辑器页禁止持久左侧导航栏。
2. 普通用户页禁止默认展示代码编辑器。
3. 普通用户页禁止默认展示 terminal。
4. P0 禁止出现 WebContainer 作为用户可见功能。
5. P0 禁止出现 HMR/热更新承诺。
6. 构建新版本时禁止清空当前稳定预览。
7. 构建失败时禁止覆盖当前版本。
8. 发布前禁止模糊当前版本。
9. 版本回退禁止无确认直接执行。
10. 禁止大面积高饱和渐变和黑科技视觉。

## 15. 设计验收清单

- [ ] 顶部是否有返回仪表盘与项目菜单。

- [ ] 失败状态是否不影响当前版本。

- [ ] 状态是否同时使用颜色和文字。
- [ ] 页面文案是否面向非技术用户。
